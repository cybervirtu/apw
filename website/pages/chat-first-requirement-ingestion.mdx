    <Steps>
    ### Recognize requirement-bearing chat

    Notice when chat introduces a requirement, change, clarification, decision, task, or evidence worth keeping.

    ### Classify it

    Decide whether it is a new requirement, change request, clarification, decision, task request, exploration only, or note/evidence only.

    ### Choose the likely destination

    Map requirement material to `SPEC.md`, work items to `TODO.md`, milestone impact to `ROADMAP.md`, rationale to `DECISIONS.md`, and exploratory or evidence-only material to `JOURNAL.md`.

    ### Save the safe summary first

    When in doubt, save a bounded summary in `JOURNAL.md` so nothing important is lost.

    ### Promote deliberately

    Use orchestrator or governance when the official project story changes across files or memory types.
    </Steps>
