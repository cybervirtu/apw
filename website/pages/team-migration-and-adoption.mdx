# Team / Migration / Adoption

This section is for people who are moving beyond "what is APW?" and into "how do we adopt or extend it safely?"

## When this path matters

Use this path when:

- you want to adopt APW in an existing repo
- you want to roll APW out to a team
- you want to run a pilot before broader adoption
- you need the compatibility and migration model

## Core message

APW stays strongest when adoption is explicit and coordinated:

- one canonical framework
- one bootstrap contract
- one validator
- thin compatibility layers
- no silent migrations or tool-specific forks
