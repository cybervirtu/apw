# APW Docs Portal

This portal is the visual front door for APW.

APW helps humans and AI agents build software without losing track of project state, governance, or execution context.

<Callout type="info" emoji="🧭">
This homepage is a guided entry page. The canonical APW rules still live in the repo-root governance files and the source docs under `docs/`.
</Callout>

<Callout type="info" emoji="🛠️">
For a brand-new repo, the friendliest first command is `/path/to/apw/apw new MyProject --profile base --stack base`.
</Callout>

## Who APW is for

APW is useful for:

- beginners who want a safer way to start AI-assisted projects
- builders who want repeatable bootstrap, validation, and CI guardrails
- teams who need shared project memory instead of chat-by-chat drift
- maintainers who want one framework that works with Codex, Antigravity, and related tooling

## The APW model in one view

| Part | What it does |
| --- | --- |
| GSD governs | protects lifecycle, planning, verification, and canonical project memory in `.gsd/` |
| AGK executes | provides specialist agents, workflows, skills, and execution support in `.agent/` |
| Orchestrator synchronizes | keeps official state changes controlled instead of letting execution notes become the project plan |

## How APW works

<Steps>
### Bootstrap the workspace

Create the repo structure from anywhere in the workspace and choose the right APW profile for the project.

### Capture the first real project state

Run the guided initializer so APW can turn the idea into the first drafts of the core `.gsd` files.

### Execute scoped work

Use the right agent and workflow instead of issuing vague prompts.

### Record bounded evidence

Execution work may add evidence to `.gsd/JOURNAL.md` without rewriting the whole project memory.

### Synchronize official state

Let the orchestrator update canonical state when `STATE.md`, `ROADMAP.md`, `TODO.md`, or `DECISIONS.md` need to change.

### Validate and continue

Use validation and CI to keep the repo aligned as work continues.
</Steps>
