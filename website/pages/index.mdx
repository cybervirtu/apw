# APW Docs Portal

APW helps humans and AI agents build software without losing track of project state, governance, or execution context.

This portal is the visual front door for the APW documentation. It lives inside the same repository as the framework, so the docs stay versioned, reviewable, and maintainable in Git.

> Markdown and MDX in this repository remain the source of truth. This Nextra site is the presentation layer, not a second governance system.

This homepage is a portal wrapper: it helps people enter APW quickly, then routes them into the right beginner guides or canonical reference material.

## Who APW is for

APW is useful for:

- beginners who want a safer way to start AI-assisted projects
- builders who want repeatable bootstrap, validation, and CI guardrails
- teams who need shared project memory instead of chat-by-chat drift
- maintainers who want one framework that works with Codex, Antigravity, and related tooling

## The APW model in one view

| Part | What it does |
| --- | --- |
| GSD governs | protects lifecycle, planning, verification, and canonical project memory in `.gsd/` |
| AGK executes | provides specialist agents, workflows, skills, and execution support in `.agent/` |
| Orchestrator synchronizes | keeps official state changes controlled instead of letting execution notes become the project plan |

## How APW usually works

<Steps>
### Start from the front door

Open root `AGENTS.md`, then route into the core APW docs and the current `.gsd` state.

### Bootstrap and validate

Create the workspace, validate it immediately, and initialize canonical project memory before routine coding starts.

### Do scoped work

Use specialist agents and workflows to build, debug, test, and improve the project.

### Keep evidence bounded

Execution agents may append bounded evidence to `JOURNAL.md`, but they should not casually rewrite canonical summary files.

### Sync official state deliberately

Use an orchestrator or governance pass when `STATE.md`, `ROADMAP.md`, `TODO.md`, or `DECISIONS.md` must change.
</Steps>
