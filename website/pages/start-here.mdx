# Start Here

Use this page as the guided map for the APW docs portal.

If you are new, the goal is simple: help you understand what APW is, how it works, and what to do next without making you read every reference document up front.

<Callout type="info" emoji="🪜">
This is a beginner-facing guide. The deeper governance and source-of-truth rules still live in the repo-root APW files and the canonical docs under `docs/`.
</Callout>

<Callout type="info" emoji="✅">
If you want the shortest safe first-use path, start with [Basic Onboarding](/basic-onboarding).
</Callout>

## Recommended beginner path

<Steps>
### Learn what APW is

Read [What Is APW?](/what-is-apw) for the plain-English explanation.

### See the journey

Read [Idea To Project](/idea-to-project) to understand how APW turns a rough idea into a structured repo.

### Choose the right shape

Read [Profiles & Stack Selection](/profiles-and-stack-selection) to choose a likely APW profile and stack direction.

### See relatable project examples

Read [Real-world Examples](/real-world-examples) to map your project to a practical starting path.

### Start the repo safely

Read the repo's `docs/QUICK_START.md`, use `apw new` to create the repo cleanly, run the guided project-state initializer, and then move into [Commands & Workflows](/commands-and-workflows).
</Steps>
