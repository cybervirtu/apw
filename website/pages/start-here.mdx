# Start Here

Use this page as the guided map for the APW docs portal.

If you are new, the goal is simple: help you understand what APW is, how it works, and what to do next without making you read every reference document up front.

<Callout type="info" emoji="🪜">
This is a beginner-facing guide. The deeper governance and source-of-truth rules still live in the repo-root APW files and the canonical docs under `docs/`.
</Callout>

<Callout type="info" emoji="📚">
APW now exposes docs in three layers: Start Now, Use APW Better, and Deep Reference.
</Callout>

<Callout type="info" emoji="✅">
If you want the shortest safe first-use path, start with [Basic Onboarding](/basic-onboarding).
</Callout>

<Callout type="info" emoji="⚙️">
If APW is not installed locally yet, start with [Installation Guide](/installation-guide).
</Callout>

<Callout type="info" emoji="📋">
If you want the important commands fast, keep the repo's `COMMAND_CHEATSHEET.md` nearby as the fallback reference.
</Callout>

<Callout type="info" emoji="🪄">
APW's preferred path is chat-first request, then IDE action, then terminal fallback.
</Callout>

<Callout type="info" emoji="🌱">
For beginners, the first APW actions to remember are: Create Project, Initialize Project State, and First Run.
</Callout>

<Callout type="info" emoji="🧭">
For orientation and safe switching, think in these actions: Show Context, List Projects, Switch To Project, Switch To Framework, and Switch To Parent.
</Callout>

<Callout type="info" emoji="📍">
The simple rule: use Create Project from anywhere, then do normal project work from the downstream project root.
</Callout>

<Callout type="info" emoji="🔀">
If you lose track of the right folder, use the repo's `docs/SAFE_CONTEXT_SWITCHING.md` model and think Show Context, List Projects, or Switch To ... first.
</Callout>

<Callout type="info" emoji="🧭">
If the project is already open in your IDE and you just want the shortest next-step checklist, go to [First Run In IDE](/first-run-in-ide).
</Callout>

<Callout type="info" emoji="🧠">
If you use `/brainstorm`, do not assume chat became permanent project memory. Save useful outcomes deliberately or hand them to orchestrator sync.
</Callout>

<Callout type="info" emoji="📝">
The same APW rule applies to the other core workflows too: keep evidence safely first, then synchronize official project memory deliberately.
</Callout>

<Callout type="info" emoji="🔄">
If APW itself changes later, use Preview Upgrade before reaching for raw bootstrap. Terminal fallback: `apw upgrade-project <name-or-path> --dry-run`.
</Callout>

<Callout type="info" emoji="🛡️">
For upgrade work, think in this order: Preview Upgrade, Upgrade Project, then Validate After Upgrade.
</Callout>

## Recommended beginner path

### Level 1 — Start now

<Steps>
### Install APW locally

Use [Installation Guide](/installation-guide) if APW is not set up on your machine yet.

### Learn the shortest first-use path

Read [Basic Usage Guide](/basic-usage-guide), then use Create Project to create the repo cleanly.

### Start the repo safely

Use [Basic Onboarding](/basic-onboarding) if you want the broader beginner-safe path around the same flow.
</Steps>

<Steps>
### Open the project the right way

Use [First Run In IDE](/first-run-in-ide) once the downstream project is open.

### Keep the fast reference nearby

Use the repo's `COMMAND_CHEATSHEET.md` if you need the command names quickly.
</Steps>

### Level 2 — Use APW better

<Steps>
### Shape the project more clearly

Read [Idea To Project](/idea-to-project), [Profiles & Stack Selection](/profiles-and-stack-selection), or [Real-world Examples](/real-world-examples) only when you need that help.

### Start work more intentionally

Run Initialize Project State, then move into [Commands & Workflows](/commands-and-workflows).
</Steps>

### Level 3 — Deep reference

Use [Reference](/reference) when you need governance, compatibility, CI, validation, or maintainer-level material.

If you want the full level map in one place, go to [Documentation Levels](/documentation-levels).
