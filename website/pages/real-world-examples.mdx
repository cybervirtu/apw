# Real-world Examples

This page answers a simple question:

"Can APW help with the kind of project I actually want to build?"

<Callout type="info" emoji="đź› ď¸Ź">
The answer is usually yes. What changes is the profile, stack direction, and the first safe workflow steps.
</Callout>

## Example map

| Project idea | Likely profile | Likely stack direction | Likely first move |
| --- | --- | --- | --- |
| personal portfolio or landing page | `minimal` or `base` | static site or lightweight web frontend | bootstrap, validate, then `/design` or `/create` |
| SaaS dashboard or admin tool | `base` | frontend + backend API + database | initialize `.gsd`, then `/status`, `/design`, or `/create` |
| AI support chatbot | `base` or `advanced` | app/API layer plus focused AI feature layer | `/brainstorm`, then `/create` or `/orchestrate` |
| internal reporting tool | `base` | dashboard or internal web app + backend + database | `/create` around the first useful reporting slice |
| multi-service platform | `advanced` | monorepo or multi-service setup | `/orchestrate` early, then specialist execution |

## The APW pattern across all examples

<Steps>
### Clarify the idea

Make the first useful version small enough to reason about.

### Choose the profile and likely stack direction

Pick the smallest APW shape that still fits the project honestly.

### Bootstrap and validate

Create the workspace and confirm the repo matches the intended APW contract.

### Start scoped work

Use focused workflows, record bounded evidence, and let the orchestrator synchronize canonical state when needed.
</Steps>
