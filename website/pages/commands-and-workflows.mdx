    <Steps>
    ### Read context first

    Start from `AGENTS.md`, then read the minimum required `.gsd` context for the task.

    ### Do scoped work

    Use the workflow that matches the job instead of issuing a vague prompt.

    ### Keep execution atomic

    Use `/orchestrate` for decomposition, then let `/create`, `/debug`, and `/test` operate on one slice at a time.

    ### Keep evidence bounded

    Execution agents may append bounded evidence to `.gsd/JOURNAL.md`.

    ### Let the orchestrator synchronize official state

    Execution agents should not freely rewrite `STATE.md`, `ROADMAP.md`, `TODO.md`, or `DECISIONS.md`.

    For brainstorm-specific persistence and promotion guidance, read the repo's `docs/BRAINSTORM_PERSISTENCE_AND_PROMOTION.md`.

    For the APW-wide save/promote model across the core workflows, read the repo's `docs/WORKFLOW_PERSISTENCE_POLICY.md`.

    For the end-to-end requirement route before workflow execution begins, read the repo's `docs/CHAT_REQUIREMENTS_TO_EXECUTION_FLOW.md`.

    For the APW model behind module and workstream decomposition, read the repo's `docs/REQUIREMENT_MODULE_BREAKDOWN.md`.

    For the APW model behind bounded execution slices, read the repo's `docs/ATOMIC_IMPLEMENTATION_PLANNING.md`.
