# Idea To Project

This is the practical journey from:

- "I have an idea"
- to "I have a structured project that is safe to build"

<Callout type="info" emoji="🚧">
APW does not magically choose everything for you. It gives you a safe path so the idea becomes shared project memory instead of getting lost across chats and half-finished notes.
</Callout>

## The journey

<Steps>
### Clarify the idea

Describe the problem, who it is for, and the smallest useful first version.

### Shape early requirements

Turn the idea into something buildable by identifying the first milestone and what must happen first.

### Choose the profile and likely stack direction

Pick how much APW structure you want up front, then choose a sensible technical direction for the first useful version.

### Bootstrap and validate

Create the APW workspace and confirm it matches the intended contract before routine coding starts.

### Run guided project-state initialization

Turn the idea into first drafts of `SPEC.md`, `ROADMAP.md`, `STATE.md`, `TODO.md`, and `STACK.md` without manually hunting through the files first.

### Start implementation safely

Use scoped agents and workflows, log bounded evidence in `JOURNAL.md`, and let the orchestrator keep official state synchronized.
</Steps>

## Where APW helps most

- APW gives the project a place for scope, roadmap, current state, and next tasks instead of leaving them scattered across prompts.
- Bootstrap, validation, and canonical state sync reduce the chance that a promising idea turns into a repo nobody understands two weeks later.
- Execution agents can move quickly, but they work inside a governed memory system instead of improvising the whole project plan.
- Validation and CI keep the workspace healthy after the initial setup, not just on day one.

## Where the orchestrator fits

Execution agents do the work.

They may append bounded evidence to `JOURNAL.md`.

The orchestrator keeps the official project summary files synchronized when `STATE.md`, `ROADMAP.md`, `TODO.md`, or `DECISIONS.md` need to change.

That is how APW keeps project memory usable instead of messy.
