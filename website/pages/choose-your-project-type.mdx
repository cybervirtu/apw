# Choose Your Project Type

Use this page when your question is:

"What kind of project am I actually building?"

This helper keeps the answer practical. It is here to help you map your idea to:

- a likely project type
- a likely APW profile
- a likely stack direction
- the best next page to read

<Callout type="info" emoji="🧠">
Project type comes before stack detail. If you can name the shape of the project clearly, the rest of the APW path gets much easier.
</Callout>

## Quick chooser

| Project type | Likely profile | Likely stack direction | Best next page |
| --- | --- | --- | --- |
| landing page or portfolio | `minimal` or `base` | static site or lightweight web frontend | [Choose Your Profile](/choose-your-profile) |
| web app or SaaS dashboard | `base` | frontend + backend API + database | [Help Me Get Started](/get-started-flow) |
| backend API or service | `base` | backend service + database or integrations | [Help Me Get Started](/get-started-flow) |
| AI assistant or AI service | `base` or `advanced` | app or API layer plus a focused AI feature layer | [Real-world Examples](/real-world-examples) |
| internal tool or automation system | `base` | internal app or service + database or automation scripts | [Help Me Get Started](/get-started-flow) |
| mobile app | `base` or `advanced` | mobile client + backend API | [Profiles & Stack Selection](/profiles-and-stack-selection) |
| monorepo or multi-service platform | `advanced` | multi-service or monorepo setup | [Real-world Examples](/real-world-examples) |

## Landing page or portfolio

### Likely fit

- profile: `minimal` or `base`
- stack direction: static site or lightweight frontend

### Best next step

Start with [Choose Your Profile](/choose-your-profile), then go to [Help Me Get Started](/get-started-flow).

## Web app or SaaS dashboard

### Likely fit

- profile: `base`
- stack direction: frontend plus backend API and database

### Best next step

Go to [Help Me Get Started](/get-started-flow), then [Commands & Workflows](/commands-and-workflows) once the repo exists.

## Backend API or service

### Likely fit

- profile: `base`
- stack direction: one backend service plus database or integrations

### Best next step

Go to [Help Me Get Started](/get-started-flow) and keep the first service scope narrow.

## AI assistant or AI service

### Likely fit

- profile: `base` or `advanced`
- stack direction: normal app or API layer plus a focused AI feature layer

### Best next step

Go to [Real-world Examples](/real-world-examples), then [Choose Your Profile](/choose-your-profile).

## Internal tool or automation system

### Likely fit

- profile: `base`
- stack direction: internal app, service, or automation flow with data where needed

### Best next step

Go to [Help Me Get Started](/get-started-flow).

## Mobile app

### Likely fit

- profile: `base` or `advanced`
- stack direction: mobile client plus backend API

### Best next step

Read [Profiles & Stack Selection](/profiles-and-stack-selection) for the fuller comparison, then go to [Help Me Get Started](/get-started-flow).

## Monorepo or multi-service platform

### Likely fit

- profile: `advanced`
- stack direction: monorepo or multi-service setup

### Best next step

Read [Real-world Examples](/real-world-examples), then [Commands & Workflows](/commands-and-workflows) to understand when `/orchestrate` matters.

## Common mistakes

- treating every idea like a platform on day one
- assuming an AI feature means the whole repo must start as `advanced`
- skipping profile choice and then guessing later
- starting code before bootstrap and validation
