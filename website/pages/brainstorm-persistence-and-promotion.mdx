# Brainstorm Persistence and Promotion

This page explains how APW keeps `/brainstorm` useful without treating chat as permanent project memory automatically.

<Callout type="info" emoji="🧠">
Canonical source doc: `docs/BRAINSTORM_PERSISTENCE_AND_PROMOTION.md`
</Callout>

<Callout type="info" emoji="📝">
Safe default: save a bounded summary to `JOURNAL.md`. If the brainstorm changes official project understanding, let orchestrator synchronize the canonical files.
</Callout>

## The practical model

<Steps>
### Explore first

Use `/brainstorm` to compare options, shape scope, and reduce uncertainty.

### Save what matters

Do not leave useful outcomes floating only in chat. Save the meaningful summary deliberately.

### Use `JOURNAL.md` by default

This preserves the session safely without prematurely rewriting official state.

### Promote agreed outcomes carefully

Move concrete requirements, tasks, roadmap changes, or rationale into the right `.gsd` files deliberately.

### Use orchestrator for official sync

When multiple canonical files should change together, hand off to orchestrator.
</Steps>
