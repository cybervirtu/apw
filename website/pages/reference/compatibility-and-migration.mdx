## The compatibility hierarchy

APW uses a unified entrypoint model so multiple tools can share the same workspace without context drift.

<Steps>
### Start at `AGENTS.md`

Use the shared modern tool entrypoint at the repo root.

### Follow the core APW docs

Treat the root governance files and canonical docs as the real APW contract.

### Keep compatibility files secondary

Treat `GEMINI.md` as compatibility-only if present, not as a second governance source.

### Treat `.agents/...` as explicit migration only

A fuller `.agents/...` layout would be a coordinated APW contract change, not a silent refactor.
</Steps>

## Future migration rule

Structural changes to the APW standard must be explicit, versioned, and coordinated. This prevents silent drift where tools adopt private namespaces or mismatched folder structures.
