## The Compatibility Hierarchy

APW uses a unified entrypoint model to ensure that multiple tools can share the same workspace without context drift.

1. **`AGENTS.md`**: The shared modern tool entrypoint.
2. **Core APW Docs**: The authoritative governance source.
3. **`GEMINI.md`**: Optional backward compatibility path.
4. **`.agents/...`**: Future pipeline-style layout (explicit migration only).


## Future Migration Rule

Structural changes to the APW standard must be explicit, versioned, and coordinated. This prevents "silent drift" where tools adopt private namespaces or mismatched folder structures.
