## Practical rule

<Steps>
### Bootstrap first

Create the intended APW workspace with the right profile and stack.

### Validate before routine work

Confirm the repo matches the expected contract before ongoing execution begins.

### Re-run enforcement as the repo evolves

Validate before major phase boundaries, releases, or rollout steps so the repo stays aligned with the APW standard.
</Steps>
