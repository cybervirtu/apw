# Downstream Project Upgrade

Use this page when APW itself has been updated and you want an existing downstream APW project to receive the latest APW-managed improvements safely.

<Callout type="info" emoji="🛡️">
The key boundary is simple: APW-managed files may be refreshed, but project-owned `.gsd` memory and product code stay protected by default.
</Callout>

<Callout type="info" emoji="🔎">
The safe beginner sequence is: Preview Upgrade, then Upgrade Project, then Validate After Upgrade.
</Callout>

Canonical source doc: `docs/DOWNSTREAM_PROJECT_UPGRADE.md`

## The short version

Think in these actions:

- `APW: Preview Upgrade`
- `APW: Upgrade Project`
- `APW: Validate After Upgrade`

Use:

```bash
apw upgrade-project MyProject --dry-run
```

Then:

- review what APW will refresh automatically
- review what APW will skip by default
- run the real upgrade only when the preview looks right
- validate afterward

## What gets refreshed safely

- the shared downstream core workflow pack
- the shared downstream core agents
- the shared downstream core rules

## What stays protected

- `.gsd/SPEC.md`
- `.gsd/TODO.md`
- `.gsd/STATE.md`
- `.gsd/ROADMAP.md`
- `.gsd/DECISIONS.md`
- `.gsd/JOURNAL.md`
- `.gsd/ARCHITECTURE.md`
- `.gsd/STACK.md`
- project code and tests
