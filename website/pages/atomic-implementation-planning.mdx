    <Steps>
    ### Start from modules or workstreams

    Keep the high-level scope in `SPEC.md`, then identify the active module or workstream.

    ### Break the active module into atomic slices

    Put actionable near-term slices into `TODO.md` as bounded tasks, not vague multi-domain plans.

    ### Keep sequence visible

    Use `ROADMAP.md` for milestone order and phase placement, not for detailed execution chatter.

    ### Execute one slice at a time

    Use `/create` for new implementation, `/debug` for a failing slice, and `/test` to verify the slice before opening the next one.

    ### Use orchestrator when official memory changes

    Save bounded evidence first, then let orchestrator or governance update canonical project memory deliberately.
