    <Steps>
    ### Classify the chat

    Decide whether the input is a new requirement, requirement change, clarification, decision, task request, exploration-only note, or evidence-only note.

    ### Make the persistence choice explicit

    Save it as notes, promote it, hand it to orchestrator, or do not save it yet.

    ### Map it into project memory

    Put requirements in `SPEC.md`, next work in `TODO.md`, milestone impact in `ROADMAP.md`, rationale in `DECISIONS.md`, and notes or evidence in `JOURNAL.md`.

    ### Break large work into modules

    Use `/orchestrate` when the requirement set is too large to leave as one planning blob.

    ### Turn active modules into atomic slices

    Keep the current work small enough for one clean execution pass and one clean verification pass.

    ### Execute with the right workflow

    Use `/create`, `/debug`, and `/test` one slice at a time, then use orchestrator or governance when official state must change.
    </Steps>
