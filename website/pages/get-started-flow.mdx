## The safest beginner path

<Steps>
### Understand the project shape

Start with [Choose Your Project Type](/choose-your-project-type) if the idea is still fuzzy.

### Choose the APW profile

Use [Choose Your Profile](/choose-your-profile) to decide how much APW structure you want up front.

### Make the repo real

Read the repo's `docs/QUICK_START.md`, then use Create Project so the repo is created and validated before routine coding starts.

### Start work intentionally

Run Initialize Project State through the repo's `docs/GUIDED_PROJECT_STATE_INITIALIZATION.md` flow, then use [Commands & Workflows](/commands-and-workflows) once the repo has real `.gsd` context.
</Steps>

## What most new users should do next

If you are still unsure, this is the safest order:

1. [What Is APW?](/what-is-apw)
2. [Choose Your Project Type](/choose-your-project-type)
3. [Choose Your Profile](/choose-your-profile)
4. `docs/QUICK_START.md` in the repo
5. `APW: Create Project`
6. `APW: Initialize Project State`
7. [First Run In IDE](/first-run-in-ide)
8. [Commands & Workflows](/commands-and-workflows)

## Common mistakes

- reading only workflow docs before deciding what the project is
- overcomplicating a prototype too early
- choosing `advanced` because it sounds safer when `base` is enough
- starting implementation before bootstrap and validation
- confusing `JOURNAL.md` evidence with canonical state updates
