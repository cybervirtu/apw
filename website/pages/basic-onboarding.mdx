# Basic Onboarding

This page is the shortest beginner route into APW.

It exists to point you to the canonical first-use guide in the repo without making you hunt for it.

<Callout type="info" emoji="✅">
Canonical source doc: `docs/BASIC_ONBOARDING_PROCEDURE.md`
</Callout>

<Callout type="info" emoji="📍">
Create the repo from anywhere with `apw new`, then do day-to-day work from the downstream project root.
</Callout>

<Callout type="info" emoji="🪜">
This page belongs to `Level 1 — Start Now` in the APW documentation model.
</Callout>

## Who this is for

Use this path if you are:

- new to APW
- a beginner
- non-technical or semi-technical
- trying APW for the first time and want the shortest safe path

## What this path covers

<Steps>
### Understand APW in one minute

Get the plain-English idea before touching the deeper docs.

### Choose the project shape and profile

Figure out what you are building and choose a likely APW profile.

### Bootstrap and validate

Create the APW repo structure from anywhere in the workspace and let the wrapper validate it for you.

### Run guided project-state initialization and start work safely

Capture the project in plain language, generate the first core `.gsd` drafts, then move into workflows and implementation.

### Use orchestrator and validation as the project grows

Keep official state controlled and keep the repo aligned over time.
</Steps>
